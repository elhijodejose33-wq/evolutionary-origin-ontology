## 3. Curatorial Standard

Every future entry or documentation change should be classified:

| Classification | Meaning |
|---|---|
| `source-direct` | Direct quotation or close paraphrase of source text. |
| `source-derived` | Conceptual transformation grounded in the source framework. |
| `framework-extension` | New application of the ontology to historical or modern systems. |
| `technical-tooling` | Code or process infrastructure not philosophically source-derived. |
| `external-evidence` | Historical, legal, scientific, or technical claims requiring independent citation. |

⸻

## 4. Citation Practice

Use citations for:

- direct source-derived claims;
- historical claims about real-world systems;
- legal, medical, financial, scientific, or technical claims;
- current platform requirements;
- licensing statements;
- claims likely to change over time.

Do not cite for:

- purely internal definitions already documented in the ontology;
- original interpretive analysis clearly labeled as such;
- code comments about local repository behavior.
